Рисунок 3.2 — Последовательность операций JWT-аутентификации

Client
Клиент
Auth Server
Сервер авторизации
Database
База данных
JWT Service
Сервис JWT
1. POST /login
Client отправляет {login, password} на Auth Server
2. SELECT * FROM sotrudnik WHERE login = ?
Auth Server запрашивает данные пользователя из Database
3. Возвращает данные пользователя
Database возвращает данные пользователя Auth Server
4. Проверка пароля (bcrypt)
Auth Server проверяет пароль
5. Генерация JWT токена
Auth Server генерирует JWT токен через JWT Service
6. Возвращает токен
JWT Service возвращает токен Auth Server
7. 200 OK {token}
Auth Server возвращает токен Client
8. Сохранение токена в localStorage
Client сохраняет токен
9. GET /protected Authorization: Bearer {token}
Client отправляет запрос с токеном
10. Верификация JWT
Auth Server верифицирует JWT токен через JWT Service
11. Валидный/невалидный
JWT Service возвращает результат верификации
12. 200 OK / 401 Unauthorized
Auth Server возвращает ответ Client
