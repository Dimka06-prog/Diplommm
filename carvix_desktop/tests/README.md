# Тесты Carvix Desktop

## Структура директории

```
tests/
├── __init__.py              # Инициализация пакета тестов
├── pytest.ini               # Конфигурация pytest
├── test_permissions.py      # Тесты системы прав доступа (RBAC)
├── test_database.py         # Тесты работы с базой данных
├── test_auth.py             # Тесты авторизации и хеширования паролей
├── test_models.py           # Тесты моделей данных
└── test_api.py              # Тесты API интеграций (GIBDD)
```

## Запуск тестов

### Установка зависимостей
```bash
pip install -r ../requirements.txt
```

### Запуск всех тестов
```bash
pytest tests/
```

### Запуск конкретного файла тестов
```bash
pytest tests/test_permissions.py
```

### Запуск с подробным выводом
```bash
pytest tests/ -v
```

### Запуск только unit-тестов
```bash
pytest tests/ -m unit
```

## Описание тестов

### test_permissions.py
Тестирует систему прав доступа на основе ролей (RBAC):
- Проверка прав для разных ролей (Диспетчер, Водитель, Механик, Администратор)
- Проверка меню для разных ролей
- Проверка названий ролей

### test_database.py
Тестирует работу с базой данных:
- Создание пула соединений
- Выполнение запросов
- Получение статистики (количество ТС, водителей и т.д.)

### test_auth.py
Тестирует систему авторизации:
- Хеширование паролей с bcrypt
- Проверка паролей
- Минимальная длина пароля

### test_models.py
Тестирует структуру моделей данных:
- Модель сотрудника
- Модель транспортного средства
- Модель штрафа
- Модель смены

### test_api.py
Тестирует API интеграции:
- GIBDD API (mock)
- Система уведомлений (mock)

## Примечания

- Тесты базы данных могут пропускаться, если БД недоступна
- API тесты используют mock-данные
- Для полноценного тестирования нужна отдельная тестовая база данных
